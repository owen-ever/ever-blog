## 아스키 아트로 UI를 시각화하는 로컬 MCP

> "좋은 제품은 사용자의 일상에 자연스럽게 스며들어야 한다"

AI가 아스키 아트로 복잡한 DB 다이어그램을 그려내는 것을 보며 영감을 얻었습니다. **가장 단순한 텍스트가 가장 직관적인 설계도가 될 수 있겠다**는 확신이 들었고, 그렇게 UI 구조를 아스키 코드로 즉각 시각화해주는 로컬 MCP [ui-ascii-art](https://github.com/owen-ever/ui-ascii-art)를 만들었습니다.

---

## 왜 만들었나요?

Experience Architect로서 제품을 만드는 이의 **경험 개선**이 곧 좋은 제품의 시작점이라고 믿습니다.

화려한 디자인 툴을 켜지 않아도 익숙한 코드 에디터 안에서 아이디어를 **즉각 구체화**하는 경험.  
그것이 바로 설계와 구현의 간극을 줄이는 혁신이라고 생각했습니다.

### 기존 개발 루틴의 문제점

요즘 저의 개발 루틴은 3단계로 이루어집니다:

1. **아이디어 구상 및 구체화**
2. **구체화된 아이디어 명세화**
3. **개발 과정**

아이디어의 UI는 마지막 단계가 되어야 확인할 수 있기 때문에 실제 확인을 위해선 **모든 과정을 거치는 토큰 비용**이 필요했습니다.

하지만 ui-ascii-art를 통해 **더 적은 토큰으로 프리뷰를 확인**하고 명세를 다듬을 수 있게 되었습니다.

## 프로젝트의 발전 과정

프로젝트가 어떻게 진화했는지 간단하게 살펴보겠습니다.

### v1.0 - 초기 릴리스 (2026-01-15)

```
feat: Initial release of UX Prototyper MCP
```

처음에는 "claude-ui-prototyper"라는 이름으로 시작했습니다.  
서버가 직접 ASCII UI를 생성하는 방식이었죠.

- 대화형 설치 스크립트(`setup.sh`) 추가
- 주식 대시보드 사용 예시로 개념 검증
- Claude Code 환경 지원

### v1.x - 멀티 플랫폼 확장 (2026-01-15~16)

```
✨ feat: Introduce Gemini support, emoji calibration
```

제가 2가지 툴을 모두 사용하기 때문에 Google의 **Gemini CLI**도 추가하게 되면서 멀티 플랫폼 도구로 확장되었습니다.

### v2.0 - 아키텍처 전환 (2026-01-16)

```
♻️ refactor: LLM 가이드 기반 UI 프로토타입 생성 방식으로 변경
```

이 커밋이 프로젝트의 **핵심적인 전환점**입니다.

**기존 방식:**

- 서버가 wcwidth 라이브러리를 사용해 직접 UI를 생성
- 서버에 UI 생성 로직이 포함됨

**새로운 방식 (LLM 네이티브 설계):**

- 서버는 UI를 직접 생성하지 않음
- 대신 **시각적 너비 규칙과 레이아웃 가이드**를 LLM에게 전달
- LLM이 **창의적이고 맥락에 맞는 UI**를 직접 생성

이 변화로 인해:

- `wcwidth` 의존성 제거
- 코드 복잡도 대폭 감소
- LLM의 창의성을 최대한 활용
- 더 유연하고 맥락에 맞는 결과물

코드 품질 개선과 함께 현재의 안정 버전이 되었습니다.

## 핵심 설계 철학

### 1. LLM 네이티브 설계

룰 베이스의 UI를 "만들어주는" 것이 아닌 LLM이 UI를 "잘 그릴 수 있도록 가이드"하는 방식입니다.

```typescript
// index.ts의 핵심 - 가이드를 반환할 뿐, UI를 직접 생성하지 않음
const guide = `
## 🎨 ASCII UI 프로토타입 생성 가이드

### ⚠️ 필수 준수 사항: 시각적 너비(Visual Width) 규칙
...
`;

return {
  content: [{ type: "text", text: guide }],
};
```

### 2. Anti-Break Protocol

전각/반각 문자의 폭 차이로 인한 레이아웃 깨짐을 방지하는 규칙을 제공합니다:

| 문자 유형         | 너비 | 예시                  |
| ----------------- | ---- | --------------------- |
| 반각 (Half-width) | 1    | 영문, 숫자, 기본 기호 |
| 전각 (Full-width) | 2    | 한글, 한자, 일본어    |
| 이모지            | 2    | ✨, 📊, 🔔 등         |
| 박스 문자         | 1    | ─, │, ┌, ┐, └, ┘      |

### 3. 리소스 최적화

코드 구현 전 텍스트 기반 미리보기를 통해:

- 기획 의도 명확화
- 불필요한 개발 리소스 소모 방지
- **토큰 비용 절감**

## 설치 방법

```bash
# 1. 저장소 클론
git clone https://github.com/owen-ever/ui-ascii-art.git

# 2. 이동
cd ui-ascii-art

# 3. 실행 권한 설정
chmod +x setup.sh

# 4. 대화형 설치 실행
./setup.sh
```

터미널의 안내에 따라 **CLI 유형**과 **설치 범위**를 선택하면 모든 설정이 자동으로 완료됩니다.

## 가장 오래된 것이 가장 새롭다

1963년, ASCII가 탄생했습니다.  
60년이 지난 지금 우리는 여전히 이 128개의 문자로 소통합니다.

Figma도, Sketch도 훌륭합니다.  
하지만 때로는 **가장 원시적인 도구가 가장 빠른 길**이 됩니다.

```
┌─────────────────────────────┐
│  아이디어 → 시각화 → 검증        │
│       단 한 번의 대화로         │
└─────────────────────────────┘
```

복잡한 것을 단순하게.  
경험의 개선이 품질의 향상이라고 믿습니다.
